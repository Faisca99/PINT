Cria uma app mobile completa em Figma para a plataforma “PINT 2025 — Plataforma de Badges da Softinsa”.

Objetivo:
Desenhar uma aplicação mobile com a mesma identidade visual, vibe, linguagem visual, componentes e experiência da versão Web existente. A app deve parecer uma extensão natural da plataforma Web, não um produto diferente. Deve manter o estilo corporativo moderno, tecnológico, limpo, profissional e gamificado da Softinsa/IBM, com dashboards, cards, badges, estados de validação, progresso visual e navegação mobile fluida.

A app deve ser desenhada para mobile, principalmente iPhone 14/15 ou equivalente, frame base 390x844, com layout responsivo também adaptável a Android. Usar Auto Layout em todos os ecrãs. Criar componentes reutilizáveis, variantes, estilos de texto, estilos de cor, ícones e estados interativos.

IMPORTANTE:
A app mobile deve incluir todas as funcionalidades da plataforma Web, adaptadas ao formato mobile. Não simplificar demasiado. Deve ter ecrãs completos para os 4 perfis:
1. Consultor
2. Talent Manager
3. Service Line Leader
4. Administrador/Gestor

A experiência deve ser mobile-first, mas com a mesma estrutura funcional da Web.

Identidade visual:
- Estilo moderno, empresarial, tecnológico e limpo.
- Usar uma paleta próxima da Web: tons de azul escuro/navy, azul vivo, branco, cinzentos claros, gradientes subtis e cores de estado.
- Cores de estado:
  - Submitted: azul
  - Em Validação: roxo/azul
  - Open/Devolvido: amarelo/laranja
  - Fechado/Aprovado: verde
  - Rejeitado: vermelho
  - Expirado: cinzento/vermelho
- Cards com cantos arredondados, sombras suaves, espaçamento generoso e hierarquia clara.
- Badges devem ter visual premium: medalhas, ícones, contornos circulares, brilho subtil, níveis A/B/C/D/E bem identificados.
- Usar tipografia moderna, legível e corporativa.
- Usar ícones consistentes para badges, dashboard, candidaturas, notificações, ranking, certificados, utilizadores, configurações, relatórios, RGPD, SLA e integrações.
- Manter uma estética semelhante à plataforma Web: dashboards com cards, listas filtráveis, tabs, chips de status, botões primários azuis, botões secundários claros e modais limpos.

Estrutura de navegação mobile:
Criar uma navegação mobile profissional:
- Bottom Navigation para ações principais do perfil.
- Menu lateral/hambúrguer ou página “Mais” para funcionalidades secundárias.
- Header com saudação contextual, avatar do utilizador, sino de notificações e seletor de idioma quando relevante.
- Breadcrumbs não são necessários em mobile; usar título de página e botão voltar.
- Usar tabs horizontais em páginas com muitas categorias.
- Usar filtros compactos em bottom sheets.
- Usar modais/bottom sheets para ações como aprovar, rejeitar, devolver, confirmar logout, escolher filtros, anexar evidências e alterar idioma.

- Evitar tabelas largas; transformar tabelas em cards filtráveis.
- Usar bottom sheets para filtros e ações.
- Usar chips para estados e níveis.
- Usar tabs para separar estados.
- Usar cards expansíveis para detalhes longos.
- Usar botões fixos no fundo quando a ação principal for importante.
- Garantir contraste e acessibilidade.
- Garantir que todos os botões têm área tocável confortável.
- Criar layout clean, sem excesso de informação por ecrã.
- Sempre que houver muitos dados, criar resumo + detalhe.
- Manter a experiência visual parecida com a Web, mas adaptada a toque.
