Atualiza o design/protótipo mobile existente da plataforma “PINT 2025 — Plataforma de Badges da Softinsa”.

IMPORTANTE:
A app mobile deve ser EXCLUSIVAMENTE para o perfil CONSULTOR.
Remove, oculta ou ignora todos os fluxos e ecrãs de:
- Talent Manager
- Service Line Leader
- Administrador/Gestor

Esses perfis existem apenas na plataforma Web. A aplicação mobile é só para consultores.

Objetivo:
Criar uma frontend mobile completa, moderna e funcional em termos visuais para o perfil Consultor da plataforma de badges digitais Softinsa. A app não precisa de backend real nesta fase. Deve usar dados mockados/estáticos para demonstrar todos os fluxos, mas deve estar desenhada de forma preparada para futura ligação à API.

Stack visual pretendida:
- React Native + Expo como referência de implementação futura
- Expo Router como referência de navegação
- TypeScript como referência
- AsyncStorage para sessão futura
- Axios/API layer futura
- Expo Document Picker/Image Picker para upload futuro
- Expo Print/Sharing para certificados futuros
- Expo Notifications para push futuro
- Expo Linking para LinkedIn/deep links futuros

Mas nesta fase quero APENAS frontend/protótipo visual no Figma.

Estilo visual:
- Manter a mesma vibe da plataforma Web PINT/Softinsa.
- Design corporativo, tecnológico, moderno e limpo.
- Visual mobile premium, com cards arredondados, sombras suaves, ícones consistentes e boa hierarquia.
- Paleta principal:
  - Azul escuro: #1e4d8c
  - Cyan/acento: #0bacda
  - Verde sucesso: #2d9d6e
  - Amarelo/âmbar aviso: #f59e0b
  - Vermelho erro/rejeição: #dc2626
  - Fundo cinza claro: #f4f5f8
  - Cards brancos: #ffffff
  - Texto principal azul muito escuro: #0f1c2e
- Usar gradientes subtis azul escuro → cyan em botões principais, headers e elementos premium.
- Badges devem parecer certificados digitais premium.
- Usar tipografia moderna semelhante a IBM Plex Sans ou equivalente.
- Usar ícones modernos e consistentes.
- Frame base mobile: 390x844.
- Usar Auto Layout em todos os ecrãs.
- Criar componentes reutilizáveis e variantes.

Arquitetura da app:
A app deve ter navegação mobile simples para Consultor.

Bottom Navigation:
1. Início
2. Badges
3. Candidaturas
4. Timeline
5. Perfil

Header comum:
- Saudação contextual
- Nome/avatar do consultor
- Sino de notificações com contador
- Botão/ícone de idioma quando aplicável

Menu Perfil/Mais deve incluir:
- Meus Badges
- Conquistas
- Leaderboard
- Recomendações
- Lembretes
- Avisos
- Certificados
- Assinatura de email
- Definições
- Terminar sessão

Criar apenas ecrãs relacionados com Consultor.
